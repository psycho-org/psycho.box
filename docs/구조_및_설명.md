## 2. 라우트 그룹 설명

| 그룹 | 경로 예시 | 용도 |
|------|-----------|------|
| `(auth)` | `/login`, `/register` | 인증 전용. 사이드바·헤더 없이 중앙 카드 레이아웃 |
| `(app)` | `/workspaces`, `/workspaces/:id/*` | 로그인 후 메인 앱. AppShell(사이드바+헤더) 사용 |

- 괄호 `()` 는 라우트 그룹: URL에 포함되지 않음
- `(auth)/login` → 실제 URL은 `/login`

## 3. 주요 파일 역할

### 3.1 `src/app/globals.css`

- **@theme**: Tailwind v4 테마 변수
  - `--color-bg`, `--color-surface`, `--color-surface-2`, `--color-surface-3`
  - `--color-line`, `--color-line-2`
  - `--color-text`, `--color-text-soft`, `--color-text-dim`
  - `--color-accent`, `--color-accent-soft`, `--color-accent-dim`
  - `--color-green`, `--color-orange`, `--color-red`, `--color-blue`, `--color-yellow`
  - `--breakpoint-lg: 960px`
- **@layer base**: body 배경/글자색, a 태그 기본 스타일

### 3.2 `src/components/auth-card.tsx`

- 로그인·회원가입 등 인증 화면용 카드
- props: `title`, `subtitle`, `children`
- 다크 테마 + accent 색상 사용

### 3.3 `src/components/app-shell.tsx`

- 앱 메인 레이아웃: 사이드바 + 헤더 + 메인 영역
- props: `children`, `workspaceId?`, `workspaceName?`, `title?`, `subtitle?`
- 워크스페이스 선택 시: 스프린트 보드, 태스크, 멤버 관리 메뉴 표시
- 미선택 시: 워크스페이스 목록 링크만 표시

## 4. 라우트 매핑

| URL | 페이지 | 비고 |
|-----|--------|------|
| `/` | → `/login` 리다이렉트 | |
| `/login` | 로그인 | Auth Layout |
| `/register` | 회원가입 | Auth Layout |
| `/workspaces` | 내 워크스페이스 | App Layout |
| `/workspaces/:id` | → `/workspaces/:id/board` 리다이렉트 | |
| `/workspaces/:id/board` | 스프린트 보드 (Kanban) | App Layout |
| `/workspaces/:id/tasks` | 모든 태스크 | App Layout |
| `/workspaces/:id/members` | 멤버 관리 | App Layout |
| `/access-denied` | 접근 권한 없음 (403) | Auth Card 스타일 |

## 5. 공통 재사용 컴포넌트 제안

### 5.1 제안 배경

검색 입력, 버튼, 리스트 조회, 카드 리스트, 로드맵 아이템 등은 여러 페이지에서 반복 사용됩니다.  
공통 컴포넌트로 분리하면:

- **일관성**: @theme 기반 스타일 통일
- **유지보수**: 한 곳 수정으로 전체 반영
- **개발 속도**: 새 페이지 구현 시 조합만으로 구성 가능

### 5.2 제안 컴포넌트 목록

| 컴포넌트 | 용도 | 사용처 |
|----------|------|--------|
| **SearchInput** | 검색어 입력 (placeholder, onChange, debounce 옵션) | 태스크 필터, 멤버 검색, 프로젝트 검색 |
| **Button** | primary / secondary / ghost / danger 변형 | 폼 제출, 모달 액션, 카드 액션 |
| **ListTable** | 테이블형 리스트 (헤더 + 행, 정렬·페이징 옵션) | 태스크 리스트, 멤버 목록, 스프린트 목록 |
| **CardList** | 카드 그리드 (카드 컴포넌트 + 레이아웃) | 태스크 카드뷰, 프로젝트 카드, 워크스페이스 카드 |
| **RoadmapItem** | 로드맵 타임라인 한 항목 (날짜, 제목, 상태, 링크) | 로드맵 뷰, 스프린트 타임라인 |
| **StatusPill** | 상태 뱃지 (TODO, IN_PROGRESS, DONE 등) | 태스크, 프로젝트, 스프린트 |

### 5.3 의견

**추천합니다.** PLAN_페이지_개발_계획에서 태스크(카드/리스트), 스프린트 보드, 멤버 관리, 로드맵 뷰 등이 모두 비슷한 UI 패턴을 사용합니다.  
초기에 컴포넌트를 정의해 두면:

1. API 연동 시 데이터만 주입하면 됨
2. 디자인 변경 시 컴포넌트만 수정
3. 접근성(a11y), 키보드 네비게이션 등을 한 곳에서 관리 가능

**구현 순서 제안**: Button → StatusPill → SearchInput → ListTable → CardList → RoadmapItem  
(의존성 순서: Button/StatusPill은 다른 컴포넌트에서 사용)

### 7.1 컴포넌트별 동작

| 컴포넌트 | 동작 |
|----------|------|
| **Button** | variant(primary/secondary/ghost/danger), loading 스피너, disabled |
| **StatusPill** | 상태별 색상, onClick 시 상태 변경 가능 |
| **SearchInput** | debounce(기본 300ms), onClear, placeholder |
| **ListTable** | 정렬(클릭 시 asc/desc), 페이징, onRowClick |
| **CardList** | renderCard로 카드 렌더, onCardClick, grid columns |
| **Roadmap** | 타임라인 드롭 영역, onPeriodChange 콜백 |
| **RoadmapItem** | 드래그하여 타임라인 셀에 놓으면 기간 변경 (시작일=드롭 셀 날짜, 기간 유지) |
