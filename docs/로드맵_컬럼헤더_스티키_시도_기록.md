# 로드맵 컬럼 헤더 스티키(Sticky) 시도 기록

## 개요

Task Roadmap 컴포넌트에서 월(月) 그룹 컬럼 헤더(예: "2026년 3월", "2026년 4월")를 가로 스크롤 시 고정(sticky)하여 항상 현재 보이는 월을 표시하려는 시도의 기록이다. 최종적으로는 구현의 복잡도와 발생한 이슈로 인해 **스티키를 제거**하고 일반 스크롤 방식으로 되돌렸다.

---

## 1. 목표 동작

### 기대했던 UX

- **좌측 고정**: Title 컬럼(태스크 목록)은 가로 스크롤 시 고정
- **우측 스크롤**: 일자 컬럼(1일~30일)은 가로 스크롤
- **월 헤더 스티키**: 스크롤 위치에 따라 현재 보이는 월(3월/4월) 헤더가 좌측에 고정되어 표시

### 사용자 시나리오

- 3월·4월 로드맵을 볼 때, 4월 구간으로 스크롤하면 "4월" 헤더가 보여야 함
- 3월·4월 경계(3월 말~4월 초)에서 헤더가 한쪽으로 치우치지 않고 자연스럽게 전환되어야 함

---

## 2. 사용 기술

### CSS

- `position: sticky`
- `left: 320px` (좌측 Title 패널 너비)
- `z-index`로 레이어링

### React

- `useState` (leadingMonthIndex)
- `useRef` (scrollRef)
- `useCallback` (updateLeadingMonth)
- `useLayoutEffect` / `useEffect` (초기 계산, ResizeObserver)
- `onScroll` 이벤트
- `requestAnimationFrame` (스크롤 처리 타이밍)

### 계산 로직

- `scrollLeft`: 스크롤 컨테이너의 가로 스크롤 위치
- `leftEdgeInTimeline`: `scrollLeft - leftPanelWidth` (타임라인 내 좌측 경계)
- `refPoint`: 기준점 (leftEdge + offset)
- 월 그룹별 누적 너비로 `refPoint`가 속한 월 인덱스 계산

---

## 3. 구현 방법

### 3.1 구조

```
[스크롤 컨테이너 overflow-auto]
  └─ [콘텐츠 flex, width: 320 + totalDays*24]
       ├─ [좌측 패널 sticky left-0] Title + 태스크 목록
       └─ [우측 패널]
            ├─ [월 헤더 행] ← 스티키 시도 대상
            ├─ [일자 행]
            └─ [태스크 바]
```

### 3.2 시도한 방식들

| 방식 | 설명 | 결과 |
|------|------|------|
| **CSS sticky만** | 각 월 헤더에 `position: sticky; left: 320px` | 3월 헤더가 늘어나 4월을 가림 |
| **sticky + right: 0** | 좌우 스크롤 모두 대응 | `left`+`right` 동시 사용 시 레이아웃 깨짐 |
| **스크롤 기반 단일 헤더** | scrollLeft로 보이는 월 계산 후 한 개 헤더만 렌더 | 기준점 조정이 어려움 |

### 3.3 기준점(refPoint) 조정 이력

- **왼쪽 끝 (0%)**: 3·4월 경계에서 헤더가 우측으로 치우쳐 보임
- **중심 (50%)**: 4월 영역으로 많이 스크롤해도 3월로 남아 있음
- **오른쪽 95%**: 3월이 거의 안 보임
- **오른쪽 25%**: 4월로 전환이 늦음
- **오른쪽 10%**: 여전히 4월로 안 바뀜
- **고정 오프셋 36px**: 개선되었으나 여전히 불안정

---

## 4. 발생한 오류/이슈

### 4.1 레이아웃

1. **3월만 보이고 4월이 안 보임**
   - `left`와 `right`를 동시에 쓰면 요소가 늘어나 3월 헤더가 화면을 가득 채움

2. **리사이즈 시 일자 컬럼 깨짐**
   - `minWidth: '100%'`로 인해 창을 줄이면 콘텐츠가 압축됨
   - `minWidth` 제거, `shrink-0` 추가로 해결

3. **월 헤더가 우측에 치우쳐 보임**
   - 3·4월 경계에서 기준점이 4월 쪽이면 4월 헤더만 표시되어 우측에 몰림

### 4.2 전환 타이밍

1. **4월로 전환이 너무 늦음**
   - 기준점을 여러 값(10%, 25%, 50%, 95%, 36px)으로 바꿔도 일관된 전환 어려움

2. **3월이 너무 빨리 사라짐**
   - 95% 기준 사용 시 3월 구간에서도 4월 헤더가 표시됨

### 4.3 파싱 오류

- JSX 주석 `{/* ... */}` 사용 시 빌드 파싱 오류 발생
- 해당 주석 제거로 해결

---

## 5. 최종 결정: 스티키 제거

### 제거한 항목

- `leadingMonthIndex` state
- `scrollRef`
- `updateLeadingMonth` 콜백
- `onScroll` 핸들러
- `useLayoutEffect`, `useEffect` (ResizeObserver)
- 스크롤 기반 단일 월 헤더

### 현재 구조

- **월 그룹 헤더**: 3월, 4월 등 모든 월을 한 행에 정적으로 표시
- **일자 행**: 1~31, 1~30 등 일자 컬럼
- **스크롤**: 전체가 함께 가로 스크롤 (좌측 Title만 `sticky left-0` 유지)

---

## 6. 향후 재시도 시 참고사항

1. **테이블 레이아웃**: `<table>` + `<th sticky>` 조합이 sticky 동작에 유리할 수 있음
2. **가상화**: react-window 등으로 보이는 영역만 렌더하면 스티키 로직 단순화 가능
3. **별도 오버레이**: 스크롤 위치에 따라 별도 레이어에 월 헤더를 그리는 방식 검토
4. **스크롤 좌표 검증**: `scrollLeft`가 sticky + flex 레이아웃에서 예상과 다르게 나올 수 있으므로 디버깅 필요
