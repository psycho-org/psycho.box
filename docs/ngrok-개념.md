# ngrok: 기술 개요

ngrok이 어떤 기술인지, 왜 필요한지, 어떻게 동작하는지를 정리한다.

---

## 1. ngrok이란

**ngrok**은 로컬(localhost)에서 실행 중인 서비스를 **인터넷에 임시로 노출**시키는 **리버스 프록시 터널링 서비스**다.

- 로컬 서버(예: `localhost:3000`)를 공개 URL(예: `https://xxxx.ngrok-free.app`)로 접근 가능하게 만든다.
- 별도 배포 없이, 같은 네트워크가 아닌 외부에서도 로컬 앱에 접속할 수 있게 한다.

---

## 2. 해결하는 문제

### 2.1 로컬 서버의 한계

| 환경 | 접근 가능 범위 |
|------|----------------|
| `localhost:3000` | 본인 PC에서만 |
| `192.168.x.x:3000` 또는 `10.x.x.x:3000` | 같은 LAN(같은 Wi‑Fi/공유기) 내에서만 |

사설 IP는 인터넷에서 라우팅되지 않기 때문에, 다른 네트워크(다른 Wi‑Fi, 모바일 데이터, 다른 지역)에서는 접근할 수 없다.

### 2.2 ngrok이 하는 일

로컬 서버를 **공개 도메인 + HTTPS**로 노출시켜, 어디서든 접속 가능하게 한다.

- 데모/테스트: 팀원, 클라이언트에게 로컬 개발 버전 공유
- 웹훅 테스트: Stripe, GitHub 등 외부 서비스가 로컬로 콜백 보내기
- 모바일 기기에서 로컬 API 테스트

---

## 3. 동작 원리

### 3.1 아키텍처

```
[외부 클라이언트]  →  [ngrok 클라우드]  →  [ngrok 에이전트]  →  [로컬 서버]
   (브라우저 등)        (공개 URL)          (사용자 PC)         (localhost:3000)
```

1. **ngrok 에이전트**: 사용자 PC에서 `ngrok http 3000` 실행 시, ngrok 클라우드와 **아웃바운드 터널**을 연다.
2. **ngrok 클라우드**: `https://xxxx.ngrok-free.app` 같은 공개 URL을 할당하고, 이 주소로 들어오는 요청을 받는다.
3. **터널 전달**: ngrok 클라우드가 에이전트를 통해 로컬 `localhost:3000`으로 요청을 전달한다.

### 3.2 핵심 개념: 아웃바운드 터널

- 일반 서버: 외부에서 들어오는 **인바운드** 연결을 받는다.
- 로컬 PC: 공유기/NAT 뒤에 있어서, 외부에서 직접 접근할 수 없다.

ngrok은 **반대 방향**으로 동작한다.

- 에이전트가 ngrok 서버에 **먼저** 연결을 연다(아웃바운드).
- 이 연결을 통해 ngrok 서버 → 에이전트 → 로컬 서버로 요청이 전달된다.
- 포트 포워딩이나 공유기 설정 없이도 동작한다.

### 3.3 기술 스택

- **프로토콜**: HTTP/HTTPS, TCP 등
- **보안**: TLS로 터널 암호화, 공개 URL은 HTTPS
- **인증**: authtoken으로 계정 연동

---

## 4. 관련 개념

| 개념 | 설명 |
|------|------|
| **리버스 프록시** | 클라이언트 요청을 받아 다른 서버(로컬)로 전달 |
| **터널링** | 한 연결 안에 다른 프로토콜/트래픽을 실어 나르는 방식 |
| **NAT traversal** | 사설망 안의 기기가 외부와 통신할 수 있게 하는 기법 |

ngrok은 이들을 조합해, 로컬 서버를 인터넷에 안전하게 노출한다.

---

## 5. 대안

| 서비스 | 특징 |
|--------|------|
| **localtunnel** | 오픈소스, 무료, ngrok보다 단순 |
| **Cloudflare Tunnel** | Cloudflare 기반, 무료 티어 |
| **Tailscale Funnel** | Tailscale 네트워크 내/외부 노출 |
| **포트 포워딩** | 공유기에서 직접 설정, 보안·관리 부담 |

---

## 6. 한계 및 주의사항

- **무료 플랜**: URL이 세션마다 바뀌고, 트래픽/연결 수 제한
- **지연**: ngrok 서버를 경유하므로 직접 접속보다 지연 증가
- **보안**: 로컬 서버가 일시적으로 공개되므로, 민감한 데이터/환경에는 신중히 사용
- **테스트 용도**: 개발·데모용으로 적합하고, 프로덕션 대체는 아님
