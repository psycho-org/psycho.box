# PLAN: 레이아웃 및 메뉴 구조

psycho.box의 화면 레이아웃과 메뉴 구성을 정의합니다.  
참고: PlanIt UI (다크 테마, 보라색 액센트)

---

## 1. 전체 레이아웃 구조

```
┌─────────────────────────────────────────────────────────────────────────────┐

├─────────────────────────────────────────────────────────────────────────────┤
│  Sidebar              │  Main Content                                        │
│  ┌─────────────────┐  │                                                      │
│  │ 워크스페이스 카드 │  │  (페이지별 컨텐츠)                                    │
│  │  DT  DevTeam     │  │                                                      │
│  │      Pro Plan ▼  │  │                                                      │
│  ├─────────────────┤  │                                                      │
│  │ 메인             │  │                                                      │
│  │  워크스페이스     │  │                                                      │
│  │  스프린트 보드 3  │  │                                                      │
```

## 2. Header (상단 바)

| 요소 | 설명 |
|------|------|
| **로고** | 좌측, 앱 브랜드 |
| **브레드크럼** | `워크스페이스명 > 현재 페이지` (예: DevTeam Alpha > 스프린트 보드) |
| **전역 액션** | 우측 고정: `+ 멤버 초대`, `+ 태스크 생성` |

---

## 3. Sidebar (좌측 네비게이션)

### 3.1 워크스페이스 카드 (상단)

| 요소 | 설명 |
|------|------|
| 아바타 | 워크스페이스 이니셜 (예: DT) |
| 이름 | 워크스페이스명 (예: DevTeam Alpha) |
| 플랜 | Pro Plan 등 (선택) |
| 드롭다운 | 워크스페이스 전환, 설정 진입 |

- **선택된 워크스페이스 ID**는 로그인 정보와 같이 **전역 상태**로 관리 ([PLAN_멀티테넌시_전역상태.md](./PLAN_멀티테넌시_전역상태.md))

### 3.2 메인 메뉴

| 메뉴 | 경로 | 설명 |
|------|------|------|
| 워크스페이스 | `/workspaces` | 내 워크스페이스 목록, 통계 카드 |
| 스프린트 보드 | `/workspaces/[id]/board` | Kanban 보드 (할일/진행중/검토중/완료) |
| 태스크 | `/workspaces/[id]/tasks` | 모든 태스크 (카드/리스트 뷰 전환, 백로그/로드맵/내 태스크 등) |
| 멤버 관리 | `/workspaces/[id]/members` | 멤버 목록, 초대 |

- 스프린트 보드: 뱃지로 활성 스프린트 수 표시 (예: 3)
- 멤버 관리: 알림 점 표시 (초대 대기 등)

### 3.3 프로젝트 섹션

| 요소 | 설명 |
|------|------|
| 프로젝트 목록 | 색상 점 + 프로젝트명 (클릭 시 해당 프로젝트 필터/뷰) |
| + 프로젝트 추가 | 프로젝트 생성 |

- 프로젝트: 웹앱 리뉴얼, API 서버 개발, 디자인 시스템, 모바일 앱 등

### 3.4 사용자 프로필 (하단)

| 요소 | 설명 |
|------|------|
| 아바타 | 사용자 이니셜 |
| 이름 | 사용자명 |
| 이메일 | 로그인 이메일 |

---

## 4. 레이아웃 종류

| 레이아웃 | 경로 | 용도 |
|----------|------|------|
| **Auth Layout** | `/login`, `/register` | 로그인/회원가입 (사이드바 없음) |
| **App Layout** | `/workspaces`, `/workspaces/[id]/*` | 로그인 후 메인 앱 (사이드바 + 헤더) |

---

## 5. 메뉴 트리 (시각화)

```
워크스페이스                 /workspaces              ← 내 워크스페이스 목록
  └─ [워크스페이스 선택 시]
       ├─ 스프린트 보드      /workspaces/[id]/board   ← Kanban
       ├─ 태스크             /workspaces/[id]/tasks   ← 테이블 뷰
       └─ 멤버 관리          /workspaces/[id]/members

[사이드바 프로젝트]
  └─ 프로젝트 클릭 시        /workspaces/[id]/board?project=xxx
                             /workspaces/[id]/tasks?project=xxx
```

## 6. 참고 UI 상세

### 6.1 내 워크스페이스 페이지

- 통계 카드: 참여 워크스페이스, 활성 프로젝트, 완료된 태스크, 팀원 수
- 워크스페이스 카드: 아바타, 이름, 역할(오너/멤버), 프로젝트/태스크/멤버 수, 멤버 아바타, 활동 상태
- + 새 워크스페이스 생성 카드

### 6.2 스프린트 보드 페이지

- 컨텍스트: `프로젝트명 2024.01.15 - 01.29`
- 액션: + 스프린트 생성, 내보내기
- 필터: 전체, 내 태스크, 높은 우선순위, 마감 임박
- 스프린트 진행: 진행률 바 (62% 8/13), 스프린트 완료 버튼
- Kanban 컬럼: 할일, 진행중, 검토중, 완료
- 태스크 카드: 타입(Feature/Design/Bug), 제목, ID(WA-042), 담당자 아바타, 우선순위
- 컬럼 하단: + 태스크 추가

### 6.3 모든 태스크 페이지

- **보기 타입**: 카드형 / 리스트형 전환
- **필터·보기**: 전체, 내 태스크, 백로그, 로드맵, 완료됨
- **카드형**: 그리드/카드 레이아웃
- **리스트형**: 테이블 (ID, 태스크, 담당자, 스프린트, 우선순위, 상태)

### 6.4 멤버 관리 페이지

- 초대 폼: 이메일, 역할, 초대 보내기
- 멤버 목록: 아바타, 이름, 이메일, 역할(오너/멤버), 설정/취소 버튼
- 초대 대기중 표시
